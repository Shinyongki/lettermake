## 배경

외부 스타일 변환 시 마크다운 표(`| ... |`)가 무시되고
본문에서 해당 섹션 내용이 통째로 빠지는 문제가 확인됨.

재현 패턴:
```markdown
## Ⅲ. 추진 일정
| 구분 | 1분기 | 2분기 | 담당 |
|------|-------|-------|------|
| 전략 수립 | 완료 | 점검 | 기획조정팀 |
| 시스템 고도화 | 착수 | 완료 | 정보화팀 |
```

기대 결과: 섹션바 아래 hwpx 테이블로 삽입  
실제 결과: 섹션바만 있고 표 내용 없음

## 구현 내용

### 1. parse_markdown_sections() — 표 파싱 추가

섹션 내 `| ... |` 패턴을 감지해서 아래 형식으로 수집합니다.

```python
{
  "type": "table",
  "headers": ["구분", "1분기", "2분기", "담당"],
  "rows": [
    ["전략 수립", "완료", "점검", "기획조정팀"],
    ["시스템 고도화", "착수", "완료", "정보화팀"],
  ]
}
```

구분선(`|---|---|`)은 건너뜁니다.
표는 슬롯 항목과 순서를 유지해서 삽입합니다 (표 앞뒤에 슬롯이 있어도 정상 처리).

### 2. build_part2() — 테이블 XML 생성 추가

`type: "table"` 항목을 만나면 hwpx 테이블 XML을 생성해서 본문에 삽입합니다.

테이블 스타일 기준:
- 헤더 행: 배경색 `D9D9D9`, 굵게, 가운데 정렬
- 데이터 행: 배경 없음, 왼쪽 정렬
- 열 너비: 내용 길이 기준 자동 배분 (전체 너비는 본문 영역 기준)
- 테두리: `000000`, `0.12mm`
- 폰트: 템플릿 기본 폰트 사용 (charPrIDRef=18 기준)

### 3. 열 너비 자동 배분

기존 방식(`markdown_parser.py`의 `_estimate_col_widths()`)과
동일한 로직을 사용합니다. 재사용 또는 참조.
