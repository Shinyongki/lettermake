## 배경

외부 스타일 변환 시 마크다운 들여쓰기 2칸(○ 슬롯) 항목이
□ 슬롯으로 잘못 처리되는 문제가 확인됨.

재현 패턴:
```markdown
- 전년도 성과 요약
  - 주요 사업 목표 달성률 94% 기록
    - 예산 집행률 98.2%로 전년 대비 3.1%p 향상
  - 민원 처리 평균 소요일 2.3일 단축
    - 온라인 접수 시스템 도입 효과
```

기대 결과:
```

  ○ 주요 사업 목표 달성률 94% 기록
```

## 작업 순서

### 1단계 — 파싱 결과 확인 (수정 전)

아래 마크다운을 `parse_markdown_sections()`에 직접 입력해서
반환되는 트리 구조를 출력하고 기대값과 비교합니다.

```markdown
## Ⅰ. 추진 배경
- 전년도 성과 요약
  - 주요 사업 목표 달성률 94% 기록
    - 예산 집행률 98.2%로 전년 대비 3.1%p 향상
  - 민원 처리 평균 소요일 2.3일 단축
    - 온라인 접수 시스템 도입 효과
```

기대하는 트리:
```
section: Ⅰ. 추진 배경
  square:  전년도 성과 요약
    circle:  주요 사업 목표 달성률 94% 기록
      dash:  예산 집행률 98.2%로 전년 대비 3.1%p 향상
    circle:  민원 처리 평균 소요일 2.3일 단축
      dash:  온라인 접수 시스템 도입 효과
```

파싱 결과가 기대값과 다르면 원인 분석 후 2단계 진행.
파싱 결과가 기대값과 같으면 `_flatten_items()` 또는
`build_part2()`의 슬롯 선택 로직을 점검.

### 2단계 — 원인 수정

1단계에서 찾은 원인을 수정합니다.
들여쓰기 레벨 감지는 공백 수 기준으로 처리합니다:
- 0칸: square (□)
- 2칸: circle (○)
- 4칸: dash (-)
- 6칸: dot (•)

탭 문자(`\t`)가 섞인 경우도 처리합니다 (탭 1개 = 공백 2칸으로 정규화).
